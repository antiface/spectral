spectral.supervisor.websocket.websocket_control module¶

class spectral.supervisor.websocket.websocket_control.ServerProtocolControl[source]¶

Bases: autobahn.twisted.websocket.WebSocketServerProtocol

WebSocket protocol for processing configuration data. The protocol describes how various connection events should be handled so that each connected client is able to modify system settings and gets notified of changes that other clients make.

Handles the WebSocket onConnect event. Fired when the client starts to connect.

Parameters:request – The request object.
onMessage(payload, isBinary)[source]¶

Handles the connection onMessage event. Fired when the server receives a message. The server will decode the message, update the new settings and relay it to all other clients.

Parameters:payload – The message content.

isBinary: Whether the payload is binary encoded.

onOpen()[source]¶

Handles the WebSocket onOpen event. Fired when the connection is established. The protocol will register itself with the factory and push a list of current settings to the client.
