spectral.supervisor.websocket.websocket_control module¶

class spectral.supervisor.websocket.websocket_control.ServerProtocolControl[source]¶

Bases: autobahn.twisted.websocket.WebSocketServerProtocol

WebSocket protocol for processing configuration data. The protocol describes how various connection events should be handled so that each connected is able to modify system settings and gets notified of changes that other clients make.

factory = None¶
onClose(wasClean, code, reason)[source]¶

Handles the connection onClose event. Fired when the connection is closed.

onConnect(request)[source]¶

Handles the WebSocket onConnect event. Fired when the client starts to connect.

onMessage(payload, isBinary)[source]¶

Handles the connection onMessage event. Fired when the server receives a message. The server will decode the message, update the new settings and relay it to all other clients.

onOpen()[source]¶

Handles the WebSocket onOpen event. Fired when the connection is established. The protocol will register itself with the factory and push a list of current settings to the client.

Relays the given message to all registered clients.

Args: client: The client that originally received the message to relay. msg: The message to relay.

Registers the given client, so that it will stop receiving updated settings.

Args: client: The :class:ServerProtocolControl instance of the client that connected.
